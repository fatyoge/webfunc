Webfunc 使用指南

浏览器自动化 + LLM 办公助手框架 — 将操作录制为 Skill，支持 CLI 执行和 MCP 大模型调用

录制 Skill 执行 Skill 打包分享 MCP Server Skill 格式 常见问题
功能概览
  • 录制 (record) — 通过 CDP 拦截浏览器请求，将操作录制为 Skill
  • 执行 (run) — 复现录制的 HTTP 流程，支持参数注入和数据提取
  • Skill 包 (package) — 自包含目录，支持自定义后置处理逻辑（TypeScript）
  • Registry 管理 — 批量管理团队 Skill 集合，一键安装全部
  • 浏览器模式 — 在已登录浏览器内执行 fetch，自动维持 session
  • HTML 解析 — 自动将 HTML 表格响应解析为结构化数据
  • MCP Server — 将 Skills 注册为 MCP Tools，供 Claude/GPT/Cursor 等大模型调用
一、录制 Skill
1
环境准备：启动浏览器

Webfunc 需要通过 CDP（Chrome DevTools Protocol）连接浏览器。

方案 A：使用已有的 Chrome/Edge（推荐，保留登录态）

关闭所有浏览器窗口后，用以下命令启动带远程调试的浏览器：

# Windows Edge
"C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" ^
  --remote-debugging-port=9222 ^
  --user-data-dir="C:\temp\chrome-dev"

# Windows Chrome
"C:\Program Files\Google\Chrome\Application\chrome.exe" ^
  --remote-debugging-port=9222 ^
  --user-data-dir="C:\temp\chrome-dev"

# macOS
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
  --remote-debugging-port=9222 \
  --user-data-dir="/tmp/chrome-dev"

# Linux
google-chrome \
  --remote-debugging-port=9222 \
  --user-data-dir="/tmp/chrome-dev"
提示：方案 A 能保留你已登录的状态（Cookie），后续执行和 MCP 调用都能直接复用。
2
执行录制命令

在项目目录下执行录制命令：

node bin/webfunc record <skill-name> -o <目标域名> -p <浏览器数据目录>

以知乎热榜为例：

# 进入项目目录
cd d:\claude\webfunc

# 执行录制
node bin/webfunc record zhihu-hot -o https://www.zhihu.com -p "C:\temp\chrome-dev"
参数说明示例
skill-nameSkill 名称，会生成 {name}.jsonzhihu-hot
-o, --origin目标域名，只录制该域名的请求https://www.zhihu.com
-p, --profile浏览器用户数据目录（可选但推荐）C:\temp\chrome-dev
-d, --dir输出目录（默认 ./skills）./skills

执行后终端会显示：

Connecting to browser...
Connected. Perform your actions in the browser. Press Enter when done.
注意：如果看到 CDP connection failed 并且没有 userDataDir 报错，说明需要添加 -p 参数。
3
在浏览器中执行操作

此时浏览器已经打开，执行你想要录制的操作流程：

  1. 在浏览器地址栏输入目标 URL 并访问
  2. 等待页面完全加载
  3. 执行目标操作（点击、填写表单等）
  4. 不要点击其他无关链接，避免录制过多噪声请求
关键提示：知乎等网站有反爬机制，录制时保持浏览器非最小化，且不要频繁操作。建议只访问目标页面，等待 3-5 秒后再停止。
4
停止录制并精简

操作完成后，回到终端窗口，按 Enter 键停止录制。

你会看到类似输出：

Skill saved to ./skills/zhihu-hot.json
Recorded 15 steps

录制会捕获所有请求，包含大量噪声（广告、统计、token 刷新等）。需要手动精简：

  • 打开 skills/xxx.json，分析每个 step
  • 只保留获取目标数据的那个 API 请求
  • 添加 parameters 定义可参数化的变量
  • 添加 extract 用 JSONPath 提取数据
  • 添加 output.summary 执行摘要

精简后的 Skill 示例：

{
  "name": "zhihu-hot",
  "version": "1.0.0",
  "description": "获取知乎热榜数据",
  "target_origin": "https://www.zhihu.com",
  "parameters": {
    "limit": {
      "type": "number",
      "default": 50,
      "description": "获取的热榜条目数量"
    }
  },
  "steps": [
    {
      "id": "hot_search",
      "method": "GET",
      "url": "https://www.zhihu.com/api/v4/search/hot_search",
      "headers": {
        "x-requested-with": "fetch",
        "user-agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
      },
      "extract": {
        "hotList": "$.hot_search_queries"
      }
    }
  ],
  "output": {
    "summary": "获取到 {{hotList.length}} 条知乎热榜数据",
    "extract": {
      "hotList": "$.hot_search_queries"
    }
  }
}

三、打包与分享 Skill
8
将纯 JSON 升级为 Skill 包

录制生成的 .json 是纯配置，无法自定义处理逻辑。升级为包格式后，每个 Skill 是一个自包含目录：

skill-zhihu-hot/ ├── skill.json # 元信息、参数定义、HTTP steps ├── index.ts # 自定义执行逻辑（可选） ├── assets/ # 模板文件等（可选） └── README.md # 使用说明（可选）

升级步骤：

# 1. 将 JSON 转为目录格式
mkdir skills/zhihu-hot
mv skills/zhihu-hot.json skills/zhihu-hot/skill.json

# 2. 创建 index.ts 自定义后置处理
cat > skills/zhihu-hot/index.ts << 'EOF'
import fs from 'fs/promises';

export default {
  async postProcess(result: any) {
    // 生成 Markdown、发邮件、调其他 API...
    const hotList = result.extracted?.hotList;
    let md = '# 知乎热榜\\n\\n';
    hotList.forEach((item: any, i: number) => {
      md += `${i + 1}. ${item.query}\\n`;
    });
    await fs.writeFile('zhihu-hot.md', md);
    return { ...result, summary: result.summary + '\\n已生成 Markdown' };
  },
};
EOF
包格式优势：
  • 后置处理逻辑写在 index.ts 中，不再依赖 webfunc 内置的硬编码处理器
  • Skill 可以独立分享给同事，安装后立即可用
  • 任意 Node.js / TypeScript 逻辑都可以写（写文件、发邮件、调其他 API）
9
发布 Skill

将 Skill 目录推送到 Git 仓库即可发布：

# 在 Skill 目录初始化 Git 仓库
cd skill-zhihu-hot
git init
git add .
git commit -m "feat: initial zhihu-hot skill"

# 推送到远程仓库（GitHub / GitLab / 内网 Git）
git remote add origin git@github.com:yourname/skill-zhihu-hot.git
git push -u origin main

同事安装时直接使用 Git URL：

node bin/webfunc install git@github.com:yourname/skill-zhihu-hot.git#main
提示：也支持本地路径分享 node bin/webfunc install ./path/to/skill，适合局域网内临时分发。
10
创建 Registry（团队 Skill 集合）

Registry 是一个 JSON 文件，列出团队所有 Skill，方便批量管理：

{
  "name": "team-skills",
  "version": "1.0.0",
  "skills": [
    {
      "name": "zhihu-hot",
      "version": "1.0.0",
      "description": "获取知乎热榜",
      "source": "git@github.com:yourname/skill-zhihu-hot.git#main"
    },
    {
      "name": "horae-failed-tasks",
      "version": "1.2.0",
      "description": "查询 Horae 失败任务",
      "source": "git@github.com:yourname/skill-horae.git#main"
    }
  ]
}

将 registry.json 放在团队可访问的位置（Git 仓库、共享目录等），然后添加：

# 添加 registry
node bin/webfunc registry add team-skills ./team-skills/registry.json

# 查看已注册的 registry
node bin/webfunc registry list

# 移除 registry
node bin/webfunc registry remove team-skills
11
安装、更新与移除 Skill

安装来源支持 registry 查找、本地路径、Git URL 三种方式：

# 从 registry 安装单个 skill（自动查找来源）
node bin/webfunc install zhihu-hot

# 安装 registry 中全部 skill
node bin/webfunc install --all

# 从本地路径安装
node bin/webfunc install ./skills/zhihu-hot

# 从 Git URL 安装
node bin/webfunc install git@github.com:yourname/skill-zhihu-hot.git#main

管理已安装的 Skill：

# 查看已安装 skill
node bin/webfunc skills list

# 更新单个 skill（重新拉取最新代码）
node bin/webfunc update zhihu-hot

# 更新全部已安装 skill
node bin/webfunc update --all

# 移除已安装 skill
node bin/webfunc remove zhihu-hot
加载优先级：执行 run 或启动 MCP Server 时，Skill 按以下顺序查找：
  1. ~/.webfunc/installed/.links/<name>/ — 已安装的全局 Skill
  2. ./skills/<name>/ — 项目本地包格式 Skill
  3. ./skills/<name>.json — 项目本地纯 JSON Skill
四、MCP Server（大模型调用）
12
什么是 MCP Server？

MCP（Model Context Protocol）是 Anthropic 推出的开放协议，让大模型能够调用外部工具。Webfunc 的 MCP Server 会将你的所有 Skills 自动注册为 Tools，供大模型按需调用。

MCP 的优势：
  • 通用 — 支持 Claude、GPT、Gemini 等任何 MCP 客户端
  • 零 UI 开发 — 配置好路径后，对话中自动出现 tools
  • 智能编排 — 大模型根据 tool 描述自主决定调用时机和参数
  • 本地安全 — stdio 通信，不走网络，无端口暴露风险
13
配置 Claude Desktop

打开 Claude Desktop 的配置文件（按 OS 路径）：

系统配置文件路径
macOS~/Library/Application Support/Claude/claude_desktop_config.json
Windows%APPDATA%\Claude\claude_desktop_config.json
Linux~/.config/Claude/claude_desktop_config.json

添加以下配置（替换为你的实际路径）：

{
  "mcpServers": {
    "webfunc": {
      "command": "node",
      "args": [
        "D:/claude/webfunc/bin/webfunc",
        "mcp",
        "-p", "C:/temp/chrome-dev"
      ]
    }
  }
}
注意：Windows 路径中的反斜杠 \ 在 JSON 中需改为正斜杠 / 或双反斜杠 \\。

保存后重启 Claude Desktop。在对话中，右下角会出现工具图标，表示 Skills 已加载。

五、Skill 格式速查
包格式（推荐）

自包含目录，支持自定义 TypeScript 处理逻辑：

skill-name/ ├── skill.json # 元信息、参数、steps ├── index.ts # 自定义 hooks（可选） └── README.md # 使用说明（可选）

skill.json 与纯 JSON 格式字段一致。index.ts 可导出生命周期 hooks：

export default {
  // 执行前钩子
  async beforeRun(context) { },

  // 自定义单步执行（如替换为浏览器内 fetch）
  async executeStep(step, context) {
    return { status: 200, data: {} };
  },

  // 后置处理：生成报告、写文件、发邮件等
  async postProcess(result, context) {
    return result;
  },

  // 执行后钩子（无论成功失败都会调用）
  async afterRun(result, context) { },
};
纯 JSON 格式（兼容）

单文件 skills/<name>.json，适合无需自定义逻辑的简单场景：

{
  "name": "skill-name",
  "version": "1.0.0",
  "description": "Skill 描述（MCP 中作为 tool 说明）",
  "target_origin": "https://example.com",
  "execution_mode": "browser",
  "parameters": {
    "param1": {
      "type": "string",
      "required": true,
      "default": "default-value",
      "description": "参数说明"
    }
  },
  "steps": [
    {
      "id": "step1",
      "method": "GET|POST|PUT|DELETE",
      "url": "https://example.com/api/data",
      "headers": {},
      "body": "key={{param1}}",
      "extract": { "data": "$.path.to.value" }
    }
  ],
  "output": {
    "summary": "获取到 {{data.length}} 条数据",
    "extract": { "data": "$.path" }
  }
}
字段说明
execution_modehttp（axios）或 browser（浏览器内 fetch，保留登录态）
parameters参数定义，type 可选 string/number/boolean
steps[].extractJSONPath 规则，提取响应数据到变量
output.summary摘要模板，支持 {{param}} 和 {{_stepId.response.path}}
注意：包格式的 index.ts 优先级高于纯 JSON 的 post_process。如两者同时存在，post_process 会被忽略。
六、常见问题
问题原因解决方案
CDP connection failed 没有启动远程调试浏览器 添加 -p 参数指定 userDataDir，或先启动浏览器
401 身份未验证 API 需要登录态 确保浏览器已登录，且使用 execution_mode: "browser"
提取结果为 undefined JSONPath 路径不匹配 检查 API 响应结构，更新 extract 路径
录制了太多无用请求 页面加载了大量广告/统计 精简 steps，只保留核心 API
MCP 工具不显示 配置路径错误或 JSON 语法错误 检查 claude_desktop_config.json 语法，路径用正斜杠
HTML 表格未解析 响应不是 JSON 且不是标准表格 检查 HTML 结构，当前只支持标准 <table> 标签
