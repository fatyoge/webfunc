Webfunc Skill 录制使用指南

将浏览器操作录制为可重复执行的 Skill 模板

1
环境准备：启动浏览器

Webfunc 需要通过 CDP（Chrome DevTools Protocol）连接浏览器，或者启动一个持久化的浏览器实例。

方案 A：使用已有的 Chrome（推荐，已登录状态）

关闭所有 Chrome 窗口后，用以下命令启动带远程调试的 Chrome：

# Windows
"C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" ^
  --remote-debugging-port=9222 ^
  --user-data-dir="C:\temp\chrome-dev"

方案 B：让 Webfunc 自动启动浏览器

在录制时通过 -p 参数指定用户数据目录：

# 录制时会自动启动浏览器
node bin/webfunc record zhihu-hot -o https://www.zhihu.com -p "C:\temp\chrome-dev"
提示：方案 A 能保留你已登录的状态（Cookie），方案 B 也能保留，但需要先手动登录一次。
2
执行录制命令

在项目目录下执行录制命令：

node bin/webfunc record <skill-name> -o <目标域名> -p <浏览器数据目录>

以知乎热榜为例：

# 进入项目目录
cd d:\claude\webfunc

# 执行录制
node bin/webfunc record zhihu-hot -o https://www.zhihu.com -p "C:\temp\chrome-dev"
参数说明示例
skill-nameSkill 名称，会生成 {name}.jsonzhihu-hot
-o, --origin目标域名，只录制该域名的请求https://www.zhihu.com
-p, --profile浏览器用户数据目录（可选但推荐）C:\temp\chrome-dev
-d, --dir输出目录（默认 ./skills）./skills

执行后终端会显示：

Connecting to browser...
Connected. Perform your actions in the browser. Press Enter when done.
注意：如果看到 CDP connection failed 并且没有 userDataDir 报错，说明需要添加 -p 参数。
3
在浏览器中执行操作

此时浏览器已经打开，执行你想要录制的操作流程：

  1. 在浏览器地址栏输入 https://www.zhihu.com/hot 并访问
  2. 等待页面完全加载（看到热榜内容）
  3. 如果需要滚动加载更多内容，可以滚动页面
  4. 不要点击其他无关链接，避免录制过多噪声请求
关键提示：知乎等网站有反爬机制，录制时保持浏览器非最小化，且不要频繁操作。建议只访问目标页面，等待 3-5 秒后再停止。
4
停止录制

操作完成后，回到终端窗口，按 Enter 键停止录制。

你会看到类似输出：

Skill saved to ./skills/zhihu-hot.json
Recorded 15 steps

录制的 Skill 文件会保存在 skills/zhihu-hot.json。

5
精简录制的 Skill（关键步骤）

录制会捕获所有请求，包含大量噪声（广告、统计、token 刷新等）。需要手动精简：

打开录制的 JSON 文件，分析每个 step，只保留获取目标数据的那个 API 请求。

以知乎热榜为例，录制的 15 个 steps 中，只有一个是真正获取热榜数据的：

{
  "id": "step_9",
  "method": "GET",
  "url": "https://www.zhihu.com/api/v4/search/hot_search",
  ...
}

精简后的 Skill 结构：

{
  "name": "zhihu-hot",
  "version": "1.0.0",
  "description": "获取知乎热榜数据并生成 Markdown",
  "target_origin": "https://www.zhihu.com",
  "parameters": {
    "limit": {
      "type": "number",
      "required": false,
      "default": 50,
      "description": "获取的热榜条目数量"
    }
  },
  "steps": [
    {
      "id": "hot_search",
      "method": "GET",
      "url": "https://www.zhihu.com/api/v4/search/hot_search",
      "headers": {
        "x-requested-with": "fetch",
        "user-agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
      },
      "extract": {
        "hotList": "$.hot_search_queries"
      }
    }
  ],
  "output": {
    "summary": "获取到 {{hotList.length}} 条知乎热榜数据",
    "extract": {
      "hotList": "$.hot_search_queries"
    }
  },
  "post_process": "generateMarkdown"
}

需要添加的字段：

  • parameters — 定义可参数化的变量（如 limit）
  • steps[].extract — 用 JSONPath 从响应中提取数据
  • output.summary — 执行后的总结文案（支持 {{var}} 模板）
  • output.extract — 最终输出时再次提取的数据
  • post_process — 后处理器名称（如 generateMarkdown）
6
测试执行

精简完成后，执行 Skill 测试：

node bin/webfunc run zhihu-hot -p "C:\temp\chrome-dev"

如果一切正常，会看到：

Connected. Getting cookies...
Executing skill: zhihu-hot

Step: hot_search - https://www.zhihu.com/api/v4/search/hot_search
  Status: 200

Summary: 获取到 50 条知乎热榜数据
Markdown saved to: d:\claude\webfunc\zhihu-hot-1234567890.md
成功！ Markdown 文件已生成，包含格式化的热榜数据表格。
7
常见问题排查
问题原因解决方案
CDP connection failed 没有启动远程调试浏览器 添加 -p 参数指定 userDataDir
401 身份未验证 API 需要登录态 确保浏览器已登录，且 Cookie 有效
提取结果为 undefined JSONPath 路径不匹配 检查 API 响应结构，更新 extract 路径
{{hotList.length}} 未替换 模板语法不支持 确保使用 {{var.path}} 格式
录制了太多无用请求 页面加载了大量广告/统计 精简 steps，只保留核心 API
